제약조건
- PK(PRIMARY KEY): 고유한 값, 중복이 없고 NULL값을 허용하지 않는다.
                         TABLE당 하나만 지정가능한 제약조건
- NULL: 아직 어떤 값을 넣을 지 모르겠다는 뜻을 가진 값
- UNIQUE KEY: NULL값은 허용하지만 중복은 허용하지 않는 제약조건
- FK(FOREIGN KEY): 외래키라고도 하며, 다른 TABLE의 PK를 의미한다.
		보통 TABLE끼리 관계를 맺을 때 사용한다.
		중복된 데이터가 들어올 수 있다.
		PK에 지정된 열에 없는 데이터를 FK로 지정된
		열에 데이터로 추가할 수 없다.

SQL문
- DDL(Data Definition Language): 데이터 정의어
    - 테이블 조작, 제어와 관련된 Query문
        1. CREATE: 테이블 생성과 관련된 명령어
        2. DROP: 테이블 삭제와 관련된 명령어 ※ 삭제 후 복구불가
        3. ALTER: 테이블 수정과 관련된 명령어
	- 테이블명 수정: RENAME TO 새로운 테이블명
	- 컬럼(열) 추가: ADD( 새로운 컬럼명 컬럼 타입 )
	- 컬럼명 변경: RENAME COLUMN 기존이름 TO 새로운 이름 
	- 컬럼 삭제: DROP COLUMN 컬럼명
        4. TRUNCATE: 테이블 내용 전체 삭제
- DML
- TCL
- DCL

자료형
- 숫자 NUMBER(자리수): 정수
         NUMBER(자리수, 소수점 자리수): 실수
         NUMBER: 자리수 생략 시 22byte까지 입력 가능(38자리 정수)	
- 문자열 CHAR(길이): 고정형, 길이가 잘 변하지 않는 것들에 사용
	ex) 주민등록번호, 전화번호
           VARCHAR2(길이): 가변형, 많은 활용을 한다.
- 날짜 DATE: format(형식)에 맞춰 날짜를 저장하는 타입
	ex) "2023.01.11"
	    "2023-01-11"
	    "2023-01"

테이블을 만들 때 유의사항
- 무결성: 데이터의 정확성, 일관성, 유효성이 유지되는 것
	- 정확성: 데이터는 애매하면 안된다.
	- 일관성: 각 사용자가 일관된 데이터를 볼 수 있도록 해야 한다.
	- 유효성: 데이터가 실제 존재하는 데이터여야 한다.
		ex)입력값이 없을 경우(모를 경우) null이라도 써라
1. 개체 무결성: 모든 테이블은 PK로 선택된 컬럼을 가져야 한다.
2. 참조 무결성: 두 테이블의 데이터가 항상 일관된 값을 가지도록 유지해야 한다.
3. 도메인 무결성: 컬럼의 타입, NULL값의 허용 등에 대한 사항을 정의하고
	          올바른 데이터가 입력되었는 지를 확인하는 것
	          (제약조건을 잘 설정하였는가)
